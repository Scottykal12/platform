# Agyn Platform Setup Deviations

This document tracks the deviations taken from the root `README.md` during the initial spin-up of the `agyn.io` instance. These changes were necessary to resolve undocumented prerequisites or environment-specific blockers.

## 1. Repository Cloning
- **README Instruction:** `gh repo clone agynio/platform`
- **Deviation:** Used `git clone` instead of `gh`.
- **Reason:** The `gh` (GitHub CLI) was not installed. Additionally, a certificate trust issue (`SEC_E_UNTRUSTED_ROOT`) required disabling SSL verification.

**Linux/Bash:**
```bash
git -c http.sslVerify=false clone https://github.com/agynio/platform.git platform
```

**Windows/PowerShell:**
```powershell
git -c http.sslVerify=false clone https://github.com/agynio/platform.git platform
```

## 2. Docker Service Redaction
- **README Instruction:** `docker compose up -d`
- **Deviation:** Commented out the `notifications` and `notifications-gateway` services in `docker-compose.yml`.
- **Reason:** These services utilize private images on GitHub Container Registry (`ghcr.io/agynio/...`). Without organization credentials, these pulls fail and block the infrastructure startup.

## 3. Environment Variable Syntax
- **README Instruction:** `export AGENTS_DATABASE_URL=...`
- **Deviation:** Used PowerShell syntax for Windows execution.
- **Reason:** Standard syntax differences between shells.

**Linux/Bash:**
```bash
export AGENTS_DATABASE_URL="postgresql://agents:agents@localhost:5443/agents"
pnpm --filter @agyn/platform-server exec prisma migrate deploy
```

**Windows/PowerShell:**
```powershell
$env:AGENTS_DATABASE_URL="postgresql://agents:agents@localhost:5443/agents"
pnpm --filter @agyn/platform-server exec prisma migrate deploy
```

## 4. Undocumented Prerequisite: Proto Generation
- **README Instruction:** (None / Step skipped in documentation)
- **Deviation:** Executed `npx @bufbuild/buf` to generate Protobuf/gRPC client code.
- **Reason:** The server requires these generated files to start. They are not included in the repository or generated by standard install scripts.

**Linux & Windows (Node.js required):**
```bash
npx @bufbuild/buf export buf.build/agynio/api --output .bufbuild
npx @bufbuild/buf generate .bufbuild --template buf.gen.yaml
```

## 5. Background Process Management
- **README Instruction:** `pnpm --filter @agyn/platform-server dev`
- **Deviation:** Ran processes in the background.
- **Reason:** To allow the UI and Server to run concurrently in a single terminal session.

**Linux/Bash:**
```bash
pnpm --filter @agyn/platform-server dev &
pnpm --filter @agyn/platform-ui dev &
```

**Windows/PowerShell:**
```powershell
# Background execution varies by environment; standard practice is separate terminals or:
Start-Process pnpm -ArgumentList "--filter @agyn/platform-server dev"
Start-Process pnpm -ArgumentList "--filter @agyn/platform-ui dev"
```
